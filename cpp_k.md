# 1. Указатели

Указатель — это переменная, которая хранит адрес другой переменной в памяти. Это один из ключевых инструментов языка C++, позволяющий работать напрямую с памятью.

Объявление указателя выполняется с помощью символа `*`. Например:

```cpp
int a = 10;
int* p = &a;
```

Здесь `p` хранит адрес переменной `a`. Символ `&` означает «взять адрес».

Чтобы получить значение по адресу, используется разыменование — оператор `*`:

```cpp
cout << *p; // выведет 10
```

Важно понимать разницу:  
`p` — это адрес  
`*p` — это значение по этому адресу

Указатели часто применяются при работе с массивами, функциями и динамической памятью. Также они позволяют эффективно передавать данные без копирования.

**Пример:**

```cpp
int x = 5;
int* ptr = &x;

cout << ptr << endl;   // адрес
cout << *ptr << endl;  // значение
```

# 2. Передача массива в функцию

В C++ массивы передаются в функцию не целиком, а как указатель на первый элемент. Это означает, что фактически передается адрес, а не копия массива.

**Пример функции:**

```cpp
void printArray(int arr[], int size) {
    for(int i = 0; i < size; i++) {
        cout << arr[i] << " ";
    }
}
```

**Вызов функции:**

```cpp
int arr[3] = {1, 2, 3};
printArray(arr, 3);
```

Запись `int arr[]` в параметрах функции эквивалентна `int* arr`.

Это важно, потому что любые изменения массива внутри функции будут влиять на исходный массив:

```cpp
void change(int arr[]) {
    arr[0] = 100;
}
```

После вызова значение первого элемента изменится.

Недостаток такого подхода — функция не знает размер массива, поэтому его нужно передавать отдельно.

# 3. Работа с файлами

Для работы с файлами в C++ используется библиотека `<fstream>`. Она предоставляет классы для чтения и записи.

**Основные классы:**

- `ofstream` — запись в файл  
- `ifstream` — чтение из файла  
- `fstream` — универсальный  

**Запись в файл:**

```cpp
#include <fstream>

ofstream file("test.txt");
file << "Hello, file!";
file.close();
```

Если файла не существует, он будет создан.

**Чтение из файла:**

```cpp
#include <fstream>
#include <iostream>

ifstream file("test.txt");
string text;

file >> text;
cout << text;

file.close();
```

Также можно читать строки целиком с помощью `getline`.

Важно всегда закрывать файл после работы, чтобы избежать потери данных.

# 4. Основы ООП

Объектно-ориентированное программирование — это подход, при котором программа строится вокруг объектов. Объекты объединяют данные и функции для работы с ними.

**Основные принципы ООП:**

- Инкапсуляция — скрытие внутренней реализации и защита данных  
- Наследование — создание новых классов на основе существующих  
- Полиморфизм — возможность использовать один интерфейс для разных типов  

ООП делает код более понятным, гибким и удобным для расширения.

# 5. Понятие класса

Класс — это пользовательский тип данных, который описывает свойства и поведение объектов.

**Простой пример класса:**

```cpp
class Person {
public:
    string name;
    int age;

    void sayHello() {
        cout << "Hello, my name is " << name << endl;
    }
};
```

**Создание объекта:**

```cpp
Person p;
p.name = "Alex";
p.age = 20;
p.sayHello();
```

Класс может содержать:

- переменные (поля)  
- функции (методы)  

**Модификаторы доступа:**

- `public` — доступен везде  
- `private` — доступ только внутри класса  
- `protected` — доступ внутри класса и наследников  

Обычно данные делают `private`, а доступ к ним дают через методы.

# 6. Наследование в ООП

Наследование позволяет создавать новый класс на основе уже существующего. При этом новый класс получает все свойства и методы родительского класса.

**Синтаксис:**

```cpp
class Child : public Parent { };
```

**Пример:**

```cpp
class Animal {
public:
    void speak() {
        cout << "Animal makes a sound" << endl;
    }
};

class Dog : public Animal {
public:
    void bark() {
        cout << "Dog barks" << endl;
    }
};
```

**Использование:**

```cpp
Dog d;
d.speak(); // унаследованный метод
d.bark();  // свой метод
```

Наследование помогает избежать дублирования кода и упрощает расширение программы.

**Типы наследования:**

- public — сохраняет уровни доступа  
- private — делает все унаследованные члены private  
- protected — делает их protected
