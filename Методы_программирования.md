# Методы программирования

**Методы программирования** — это совокупность подходов, принципов и способов разработки программного обеспечения, определяющих структуру программы, логику её работы и процесс создания.

---

## 1. Классификация методов программирования

Методы программирования можно классифицировать по нескольким признакам:

- по уровню абстракции  
- по организации кода  
- по стилю и парадигме  
- по способу взаимодействия с пользователем  
- по архитектуре приложений  

---

## 2. Основные методы (парадигмы) программирования

### 2.1 Процедурное программирование

Один из самых ранних и простых методов.

**Суть:**
- программа представляется как последовательность процедур (функций)
- данные и функции разделены слабо
- управление осуществляется через вызов функций

**Характерные особенности:**
- линейная структура выполнения
- активное использование условий и циклов
- общие глобальные данные

**Преимущества:**
- простота понимания
- высокая скорость выполнения
- подходит для небольших программ

**Недостатки:**
- плохая масштабируемость
- сложность поддержки больших проектов

**Примеры языков:**  
C, Pascal

---

### 2.2 Модульное программирование

Метод, основанный на разбиении программы на независимые модули.

**Суть:**
- программа состоит из логически завершённых модулей
- каждый модуль выполняет конкретную задачу
- модули можно разрабатывать и тестировать отдельно

**Преимущества:**
- повышает читаемость кода
- упрощает сопровождение
- повторное использование модулей

**Недостатки:**
- требует хорошего проектирования

**Примеры:**  
C (с заголовочными файлами), Python (модули), Java

---

### 2.3 Объектно-ориентированное программирование (ООП)

Один из наиболее распространённых методов.

**Суть:**
- программа строится из объектов
- объекты объединяют данные и методы работы с ними

**Основные принципы ООП:**
- **Инкапсуляция** — скрытие внутренней реализации
- **Наследование** — создание новых классов на основе существующих
- **Полиморфизм** — единый интерфейс для разных реализаций
- **Абстракция** — выделение важных характеристик

**Преимущества:**
- удобно для больших проектов
- повышает повторное использование кода
- лёгкое расширение функциональности

**Недостатки:**
- более высокая сложность
- потери производительности по сравнению с процедурным подходом

**Примеры языков:**  
Java, C#, C++, Python

---

### 2.4 Событийно-ориентированное программирование

Широко используется в оконных и web-приложениях.

**Суть:**
- работа программы определяется событиями
- события: нажатия кнопок, движения мыши, ввод данных

**Особенности:**
- основной цикл ожидания событий
- обработчики событий
- асинхронное выполнение

**Применение:**
- GUI-приложения
- Web-интерфейсы
- мобильные приложения

---

### 2.5 Функциональное программирование

Метод, основанный на математических функциях.

**Суть:**
- отсутствие изменяемого состояния
- функции — основные элементы программы
- минимум побочных эффектов

**Преимущества:**
- предсказуемость кода
- удобство параллельных вычислений
- высокая надёжность

**Недостатки:**
- сложность освоения
- менее интуитивно для новичков

**Примеры языков:**  
Haskell, Lisp, Scala
